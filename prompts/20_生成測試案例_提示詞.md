# 20 — 生成測試案例

## 使用時機
spec md 已產出且 review 過（USER-EDIT 區塊已填或確認無補充）。要把 AC 變成 pytest 可執行的 test 檔。

## 必填參數
- **工項清單**：例 `2-2-2-B`、`2-2-3`、`2-2-4-A`
- **spec 路徑**：`specs/<父工項> <名>/<id>.md`
- **selectors 庫**：`lib/selectors.py`（已有的 Page Object 優先重用，不夠用再加）
- **目標機制**：`tests/<父工項> <名>/test_<id>_*.py`

## 期望輸出
每個工項一份 test 檔（一個 module 多個 case，依 AC 切 function），命名 `test_<工項id以底線取代橫線>_<關鍵字>.py`。每個 test function 對應一個 AC，加 docstring 標明對應 AC 編號。

## 標準流程

### 步驟 0：前置確認
1. **讀 spec**：把 AC、待釐清、USER-EDIT、實作差異註記都看過一遍
2. **讀 lib/selectors.py**：找對應頁的 Page Object（若無，步驟 2 要加）
3. **讀 prompts/99_重點經驗.md** 最新 5 條：避開已知踩坑

### 步驟 1：解碼 src（CLAUDE.md 第 1 條第 1 步）
依 spec 對應的功能找 src：
- 列表頁類：`src/.../view/<XxxList>.vue` + `src/.../entity/<Xxx>Entity.ts` + `src/.../STFilter.vue`
- 編輯頁類：`src/.../view/<XxxEdit>.vue` + entity model
- 共用 wrapper：`STDataTable / STFilter / PageBuilder` 系列

確認：
- 元件 label 文字（grep entity model 的 `Title_*`）
- 是否有 `data-testid`（無則用語意 selector）
- 動態 ID（PrimeVue `pv_id_*`）位置 → 一律避開

### 步驟 2：補 lib/selectors.py（必要時）
新增 Page Object class：
```python
class XxxPage:
    PATH = "/entry/xxx"

    @staticmethod
    def primary_input(page: Page) -> Locator:
        # 依 src 證據寫，不是猜
        return page.locator("label").filter(has_text="主辦單位").locator("xpath=..").locator(".p-multiselect")
```

優先順序：`get_by_role` > `get_by_label` > `get_by_text` > `locator(".semantic-class")` > **絕不用** `pv_id_*` / Tailwind utility

### 步驟 3：寫 test 檔
**標準骨架**：

```python
"""
工項：<id> <標題>
規格：specs/<父工項> <名>/<id>.md

實作狀態（src 解碼 <YYYY-MM-DD>）：
- <要點 1>
- <要點 2>

測試策略：
- 「正向 AC」（合法值不被擋 + 元件存在）→ 可跑
- 「反向 AC」（違反規則時應有防呆）→ 視 src 狀態：
  - 已實作 → 直接驗
  - 未實作 → @pytest.mark.xfail(strict=False, reason="...")
"""
from __future__ import annotations
import pytest
from playwright.sync_api import Page, expect

from lib.selectors import XxxPage

from . import _shared

WBS_ID = "<id>"
pytestmark = [pytest.mark.wbs(WBS_ID), pytest.mark.module("<父工項>")]


def test_AC1_xxx存在(page: Page, config, report_attach):
    """AC1：<行為描述>"""
    _shared.open_xxx(page, config["base_url"])
    el = XxxPage.primary_input(page)
    report_attach(url=page.url, expected="<期望>")
    expect(el).to_be_visible()
    report_attach(actual="<實際觀察>")
```

### 步驟 4：寫 _shared.py（同目錄）
工項間共用的進入頁面、操作流程封裝（避免 test 檔重複）。

```python
"""<工項> 共用測試輔助。"""
from playwright.sync_api import Page
from lib.selectors import XxxPage

def open_xxx(page: Page, base_url: str) -> None:
    page.goto(base_url + XxxPage.PATH, wait_until="domcontentloaded")
    XxxPage.primary_input(page).wait_for(state="visible", timeout=15000)
    page.wait_for_timeout(1500)
```

### 步驟 5：標 xfail / skip 的判準

| 情境 | 標記 | reason 寫法 |
|---|---|---|
| spec 與 impl 衝突（例 D-AC2 disabled gate） | `@pytest.mark.xfail` | `"spec 期望 X，impl 為 Y；待業務方確認"` |
| 元件未實作（推測自舊版 src） | `@pytest.mark.xfail(strict=False)` | `"依舊版 src 推測 <X> 未實作；新版測試機可能已實作（XPASS 表示可解 xfail）"` |
| 外部依賴未到位（前置工項未完） | `@pytest.mark.skip` | `"前置 <id> 未完，無法驗證"` |
| 測試環境資料品質問題（dev 資料髒） | `@pytest.mark.xfail` | `"dev 環境資料品質問題：<X>；建議走 DB 直查"` |

**禁止**：把測試硬塞 PASS（assert True、try/except 吞錯、改 spec 對齊 impl 而沒記錄）。紅綠是事實，不是面子問題。

### 步驟 6：fixture 與 marker 約定
- `page`、`config`、`report_attach`：conftest.py / md_reporter 提供，直接用
- `pytest.mark.wbs("<id>")`：必加，報告器依此分組
- `pytest.mark.module("<父工項>")`：必加，docx 整併用

## 命名規則
- 檔名：`test_<id 以底線取代橫線>_<關鍵字>.py`，例 `test_2_2_2_B_主辦單位活動日期活動名稱篩選.py`
- 函式名：`test_AC<n>_<關鍵字>`，中文允許
- 同 AC 多分支：`test_AC<n>_<變體>`，例 `test_AC8_日期格式_slash` / `test_AC8_日期格式_dash`

## 已知踩坑（必看）
- **PrimeVue DatePicker 毛病**：`fill()` 後不會自動 commit v-model、不會收 overlay。**必須加** `press("Enter")` 或 `press("Tab")`，再 `wait_for_timeout(500)`
- **PrimeVue MultiSelect 互動 selector 未解**：C-AC2 / B-AC4 目前 xfail
- **DOM 是唯一事實**：spec 寫「套用」但 DOM 文字是「查詢」→ 以 DOM 為準
- **Playwright 1.48 sessionStorage**：conftest.py 已用 `add_init_script` 還原，test 不需自己處理
- **xfail strict=False 含義**：失敗 → XFAIL；意外通過 → XPASS（不會 fail）。XPASS 是訊號告訴你「可解 xfail」

## 完成後 handoff
1. 把 test 檔交給人類跑：`pytest "tests/<父工項> <名>/" -v`
2. 紅了 → `prompts/40_執行測試與排錯_提示詞.md`
3. 全綠 → 跑 `pytest ... --shot=always` 拍截圖 → `python tools/md_to_docx.py` 出 docx
