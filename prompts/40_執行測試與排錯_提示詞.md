# 40 — 執行測試與排錯

## 使用時機
test 已寫完，要跑 pytest；或 pytest 紅了要排查。

## 必填參數
- **目標工項或測試路徑**：例 `tests/2-2-2 搜尋功能/`
- **執行環境**：本機 Win11 + PowerShell + venv
- **失敗時可額外提供**：pytest short summary、HTML dump（user 用 explore_page.py 或 dump 工具產）

## 期望輸出
- **跑 pytest（人類動手）**：short summary + reports/<rid>_run/ 目錄
- **排錯（AI 動手）**：根因分析 + 修訂建議（落到 `lib/selectors.py` / test 檔 / spec md 三選一）

---

## 標準流程

### 階段 1：跑測試（人類執行）

**收斂中（除錯階段）**：
```powershell
pytest "tests/<父工項>/" -v
```

**最終交付（要產 docx）**：
```powershell
pytest "tests/<父工項>/" -v --shot=always
python tools/md_to_docx.py
```

**單一工項排查**：
```powershell
pytest "tests/<父工項>/test_<id>_*.py::test_<AC名>" -v
```

**只跑某類 marker**：
```powershell
pytest -v -m "wbs and not slow"
```

短輸出：
```powershell
pytest "tests/..." --tb=line -q
```

### 階段 2：解讀 short summary

| 結果 | 意義 | 下一步 |
|---|---|---|
| **PASSED** | 行為符合預期 | ✅ 收進交付清單 |
| **FAILED** | 行為不符合預期 | → 階段 3 排錯 |
| **XFAIL** | 預期失敗，實際失敗（`@xfail` 生效） | ✅ 訊號正確，記錄不解 |
| **XPASS** | 預期失敗，實際通過 | ⚠️ 解 xfail 訊號（前端可能修了） |
| **SKIPPED** | 主動跳過 | 看 reason，多是前置依賴未到位 |
| **ERROR** | setup / teardown 出錯（非 assertion） | → 階段 3 排錯（通常是 selector 找不到元件） |

> **XPASS 不是 FAIL**：strict=False 下 XPASS 不會讓 pytest 退出碼非零。但 XPASS 是「該解 xfail」的訊號，要主動處理。

### 階段 3：排錯 — selector 失敗（最常見 80%）

**4 步流程**（CLAUDE.md 第 1 條）：

#### 第 1 步：解碼 src

對應頁的 src 路徑：
- 列表頁 → `src/.../<XxxList>.vue` + `<Xxx>Entity.ts`
- 編輯頁 → `<XxxEdit>.vue` + entity model
- 篩選器 → `STFilter.vue` 系列

抓三類證據：
1. **元件 label**：搜尋 `Title_<field>` 或 `label="<text>"`
2. **元件型別**：PrimeVue 元件名（DatePicker / MultiSelect / InputText...）
3. **動態渲染**：是否包在 `v-for` 內、是否有條件渲染

#### 第 2 步：改 lib/selectors.py

依 src 證據改寫 Page Object，**不對 live DOM 試錯**。

優先順序：
```
get_by_role(name=...) 
  > get_by_label(...) 
  > get_by_text(...) 
  > locator(".semantic-class")
  >> 絕不用 pv_id_* / Tailwind utility class
```

#### 第 3 步：再跑 pytest

```powershell
pytest "tests/<父工項>/test_<id>_<AC>" -v
```

#### 第 4 步：還紅 → 請人類 dump HTML

**輕量探查（90% 用這個）**：
```powershell
python tools/explore_page.py /entry/<path>
```

列出該頁所有 button / placeholder / heading / data-testid。

**完整 HTML dump（探查不夠時）**：
```powershell
python tools/dump_runtime_storage.py
# 或
python tools/dump_storage_now.py
```

人類把輸出貼回 → AI 對著真實 DOM 改 selector。**最多 3 輪**，超過要停下來重新解碼 src。

### 階段 3'：排錯 — 不是 selector 的失敗

| 症狀 | 可能根因 | 處置 |
|---|---|---|
| Timeout 等元件 visible | DOM 渲染慢、條件渲染、登入失效 | (a) 確認 session 還有效（`tools/run.py --warm-login`）；(b) 加 `wait_for_url` 確認到對的頁；(c) timeout 加長到 15s 試 |
| `fill()` 後 v-model 沒更新 | PrimeVue DatePicker / MultiSelect 毛病 | 加 `press("Enter")` + `wait_for_timeout(500)` |
| 列數 / 內容對不上預期 | dev 環境資料變了 / spec 描述不準 | (a) 直查 DB 或人工掃 UI 確認資料；(b) 若是資料品質問題標 xfail；(c) 若是 spec 不準改 spec |
| import 錯誤 | pythonpath 沒設 | 確認 `pytest.ini` 有 `pythonpath = .` |
| `storage_state` 過期 | session_max_age 超過 | 跑 `python tools/run.py --warm-login` 重產 |
| 隨機 fail（時通時不過） | wait 不足 / 競態 | 加 `expect(...).to_be_visible(timeout=15000)` 或 `wait_for_load_state("networkidle")` |
| `--shot=always` 截不到圖 | reporter plugin 沒載入 | 確認 `conftest.py` 有 `pytest_plugins = ("lib.md_reporter",)` |

## XPASS 處理 SOP

收到 XPASS = 「該前端已實作，可解 xfail」訊號。

1. **確認**：再跑一次該 case 確認 XPASS 穩定（不是偶發）
2. **解碼新 src**（若可取得新版前端 code）：找出實作位置記到 spec 的「實作差異註記」
3. **解除 xfail**：刪 `@pytest.mark.xfail(...)` decorator
4. **重跑驗 PASS**
5. **記到 STATUS.md**：「<日期> XPASS 解 xfail：<id> AC<n>，前端已實作 <X>」

## docx 交付前檢查

```powershell
pytest "tests/<父工項>/" -v --shot=always
python tools/md_to_docx.py
```

檢查 `reports/<rid>_run/docx/<rid>_run_測試報告.docx`：
- [ ] 每個 case 一段（標題 + 結果 + 截圖）
- [ ] PASS / XFAIL / SKIP 統計表正確
- [ ] 截圖完整（沒缺）
- [ ] 中文沒亂碼

紅 → 跑 `tools/md_to_docx.py` 看 stderr，多半是 PermissionError（docx 還開在 Word，關掉重跑）。

## 反模式

- ❌ **對 live DOM 盲試 selector**：4-30 token 教訓，一天燒完額度
- ❌ **改 spec 對齊 impl 而沒記錄**：以後再 revisit 不知道為何那樣寫
- ❌ **連續紅三輪不請人類 dump HTML**：硬撐反而更花 token
- ❌ **xfail strict=True 用在「推測未實作」**：應該用 strict=False，不然測試機已實作會變 FAIL
- ❌ **`assert True` 或 `try/except: pass` 強綠**：失去測試價值
