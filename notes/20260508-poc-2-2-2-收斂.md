## 2. 初始 Prompt

> 引自 `初始提示詞.md`（節錄重點）：

```
我現在要針對一個網站開發專案導入自動化測試
範圍：WBS.md（工項編號）+ 需求規格.pdf

需求：
- 自動化工具自行操作瀏覽器測試
- 依工項編號跑指定腳本
- 自動產 md 測試報告
- 一個工項一目錄、可審查修訂
- 規格 PDF → md 展開、可審查修訂
- 提示詞與重點經驗可被記錄、跨專案重現
- Win11 + WSL2 雙環境（最後僅考慮 Win11）
- 圖形驗證碼預留繞過開關
- 我每週提交 3~5 個工項，你負責 規格展開 → 案例 → 測試 → 回報
```

## 3. 過程 Prompt 摘要（重要轉折點）

引自 `prompts/99_重點經驗.md`：

1. **2026-04-30 token 教訓**：AI 對 live DOM 盲試 selector 一天燒完額度 → 改「先讀 src 再寫 selector」四步流程（CLAUDE.md 第 1 條）
2. **2026-04-30 DOM 是唯一事實**：i18n 寫「套用」但 DOM 文字是「查詢」→ 文字 selector 不從 i18n 推斷
3. **2026-04-30 PrimeVue 動態 ID 規避**：`pv_id_*` 每次刷新都變 → 改用語意 selector（`get_by_role` / `get_by_label` / `.semantic-class`）
4. **2026-04-30 Playwright 1.48 sessionStorage workaround**：`storage_state()` 不抓 sessionStorage，用 `add_init_script` 還原
5. **2026-05-05 STATUS.md + harness scaffold**：跨 session 進度錨點落地（`harness-engineer` skill 一鍵建齊 `CLAUDE.md / plan.md / STATUS.md / docs / notes`）
6. **2026-05-07 xfail(strict=False) 策略**：本機 src 是舊版，30 個 SKIP-pending 改 xfail(strict=False)；XPASS 即「測試機已實作、可解 xfail」訊號

### 6.2 工程化骨架（一次寫好、跨專案重用）

- **`lib/`**：4 個共用模組（auth / config_loader / md_reporter / selectors）
- **`tools/`**：7 個 CLI 入口（warm_login / run / md_to_docx / explore_page / extract_pdf_text / 2 個 dump 工具）
- **`prompts/`**：5 份提示詞（00 專案發起 / 10 PDF 轉規格 / 20 生成測試案例 / 30 審查修訂 / 40 執行排錯）+ 99 重點經驗
- **`docs/`**：3 份永久參考（工具指令集 / 合作 SOP Q&A / 技術架構）

### 6.3 Token 用量

- **改採「先讀 src 再寫 selector」前**：1 天額度跑完 4 case 即耗盡（2026-04-30 教訓）
- **改採後**：1 day budget 可推進 1 個父工項（5 case）+ 多輪 review，估算節省 5–10 倍試錯成本

### 6.4 執行穩定度

- 連續 10+ 次 ad-hoc run，failure rate < 5%
- Nightly 排程未啟動（P5.7 待 P4.5 全綠）
