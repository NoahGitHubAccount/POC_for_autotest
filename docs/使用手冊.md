# 使用手冊

> **對誰寫**：實際使用本工具的 user（QA / 工程師 / AI 協作者）。
> **怎麼用**：先看 Part A 把週循環 SOP 建立心智模型 → 動手時查 Part B 找對應 CLI 指令。
> 最後更新：2026-05-08

## Part A. 每週協作 SOP

> 本段是「人怎麼跟 AI 講話」、「跨 session 接手」的互動 SOP。新人 / 新 session 從這裡開始。

### Q1：每週你（user）要給 AI 什麼？

**A**：3–5 個 WBS ID + 各工項的「實作狀態」。

範例貼法：

```
本週工項：
- 2-2-4-A 活動編號：已實作（前後端都好，可測）
- 2-2-4-B 活動名稱：已實作
- 2-2-4-G 最近更新時間排序：已實作
- 2-2-3 活動列表預設排序：已實作
備註：2-2-4-H 還沒，下週再驗
```

最少資訊量：**WBS ID + 「可測 / 未實作 / 部分（誰擋住）」**。
有額外提示（特殊 fixture、測試帳號、邊界值等）也一併寫，沒有就不寫。

---

### Q2：AI 每週給你什麼？

**A**：對每個工項依序產出 4 樣交付物：

1. **spec md**（若還沒展開）：`specs/<工項id>/<工項id> 標題.md`
2. **test 檔**：`tests/<父工項> <名>/test_<id>_*.py`
3. **pytest 結果**：你跑、你貼 short summary 給 AI
4. **docx 交付件**：`reports/<rid>_run/docx/<rid>_run_測試報告.docx`

---

### Q3：單一工項從 0 到交付的 motion path

**A**：固定 5 步。

1. **spec 沒展開** → AI 跑 `prompts/10_PDF轉規格_提示詞.md` 對該工項，產 spec md。**你 review**，把調整寫進 `<!-- USER-EDIT START -->` 區塊。
2. **寫 test** → AI 參考 spec + `lib/selectors.py`，產 `test_<id>_*.py`。標清 xfail / skip 的理由（spec/impl 衝突？元件未解碼？外部依賴？）。
3. **跑 pytest** → 你執行：
   ```powershell
   pytest "tests/<父工項> <名>/" -v
   ```
   把 short summary 貼回給 AI。
4. **紅了 → 4 步 selector 修復流程**（CLAUDE.md 第 1 條）：
   - ① 解碼 src（讀對應 view + entity model + STFilter）
   - ② 修 `lib/selectors.py`
   - ③ 再跑 pytest
   - ④ 還紅 → 你 dump 一頁 HTML（`/entry/<path>` 載入後）給 AI，AI 對著真實 DOM 改。**最多 3 輪**。
5. **全綠 → 交付**：
   ```powershell
   pytest "tests/<父工項> <名>/" -v --shot=always
   python tools/md_to_docx.py
   ```
   docx 在 `reports/<rid>_run/docx/`。

---

### Q4：截圖模式怎麼選

**A**：兩種模式：

| 階段 | 命令 | 行為 |
|---|---|---|
| **收斂中** | `pytest ...`（不加 flag） | `--shot=failed_only`（預設）：只失敗時拍，省檔案量 |
| **最終交付** | `pytest ... --shot=always` | 每個 test 都拍，docx 內每個 case 都有畫面 |

完全不需要時：`--shot=off`。

---

### Q5：AI 跨 session 怎麼接得回上次進度

**A**：每次 session 開場 AI 先讀 `STATUS.md`。session 結束若有實作或決議異動就寫回去。

`STATUS.md` 三段最重要：

- **WBS 進度表**：誰在跑 / 誰阻塞
- **最近決議**：最新在上、保留 ~10 條
- **下一步**：你看這段就知道 AI 打算做什麼

純 read-only Q&A AI **不**動 STATUS.md。

---

### Q6：你想直接看現在做到哪

**A**：開 `STATUS.md` 看「目前焦點」+「下一步」兩段。其餘段落是支撐資訊。

---

### Q7：spec 與實作衝突時怎麼處理

**A**：

- **小衝突（按鈕文字、欄位順序、提示語等）**：以 DOM 為準（CLAUDE.md 第 3 條），測試對齊實作；spec md 加註記「實作為 X，規格為 Y，已採 X」。
- **大衝突（spec 要求的行為實作直接擋掉，例：disabled gate）**：標 xfail + reason 引到「待業務方確認」；同時記到 STATUS.md「待解問題」。**不擅自改 spec**，不擅自把測試搞綠。

---

### Q8：可以一次給多週的工項嗎

**A**：可以，但建議一週批一次跑。理由：

- 開發狀態變化快，跨週的工項實作狀態可能變
- 你的 review 容量有限，一次太多會卡在 (1) spec review
- pytest 紅了要回頭 dump HTML，連續工項並行只會拉長 cycle time

**例外**：同一父工項下的子工項（如 2-2-4-A~G）屬於同一輪，可一次給。

---

### Q9：你跑 pytest 紅了 + AI 說要 dump HTML，怎麼 dump

**A**：兩個工具，看狀況挑。

**輕量探查（先試這個）**：

```powershell
python tools/explore_page.py /entry/<path>
```

列出該頁所有 button 文字、placeholder、heading、data-testid。**90% 的 selector 失敗用這個就能解**（如「按鈕到底叫什麼名字」）。

**完整 HTML dump（探查不夠時用）**：

```powershell
python tools/dump_runtime_storage.py
```

把輸出貼回，或檔案路徑告訴 AI。

---

### Q10：用什麼一句話啟動本週的工作

**A**：貼 Q1 範例那種格式，後面加一句「**動工**」。例：

```
本週工項：
- 2-2-4-A 活動編號：已實作
- 2-2-4-B 活動名稱：已實作
- ...
動工
```

AI 會：

1. 先跑 spec 展開（prompts/10_）→ 產 spec md
2. spec 給你 review
3. review 後寫 tests
4. 你跑 pytest，貼 summary
5. 全綠 → docx 交付

### B1. 產 Word 報告

#### 標準流程：跑測試 → 出 docx

```powershell
# 步驟 1：跑測試（每個 case 都截圖）
pytest --shot=always

# 步驟 2：把 reports/ 最新 run 整併成 docx
python tools/md_to_docx.py
```

輸出：`reports/<rid>_run/docx/<rid>_run_測試報告.docx`（單份檔，所有工項報告整併）。

#### 指定特定 run

```powershell
# 用 run_id（reports/ 下的目錄名前綴）
python tools/md_to_docx.py 20260506_2243_run

# 或只給時間部分也可
python tools/md_to_docx.py 20260506_2243
```

#### 沒裝 python-docx 時

```powershell
pip install python-docx==1.1.2
```

#### 細節

- 截圖「嵌入」docx（不是參照），單檔可獨立流通
- `_summary.md` 在最前，其他工項報告依檔名排序，之間插分頁符
- 重跑 docx 不需要重跑 pytest——只要對應 run 的 `*.md` + `screenshots/` 還在就能轉

---

### B2. 跑測試

#### 範圍選擇

```powershell
# 全部
pytest

# 單一工項目錄
pytest "tests/2-2-3 活動列表預設排序/" -v

# 多個工項
pytest "tests/2-2-3 活動列表預設排序/" "tests/2-2-4 活動列表欄位/" -v

# 用 wbs marker（如 2-2-2-B 全部）
pytest -m "wbs('2-2-2-B')" -v

# 只跑單一 test function
pytest "tests/2-2-2 搜尋功能/test_2_2_2_B_主辦單位活動日期活動名稱篩選.py::test_AC5_日期區間篩選" -v
```

#### 截圖模式（pytest 自訂 option）

```powershell
pytest --shot=failed_only   # 預設：只失敗拍
pytest --shot=always        # 每個 case 都拍（產交付 docx 時用這個）
pytest --shot=off           # 完全不拍
```

#### 失敗時看更多

```powershell
pytest -v --tb=short        # 簡短 traceback
pytest -v -x                # 第一個失敗就停
pytest -v --lf              # 只重跑上次失敗（last-failed）
pytest -v --ff              # 失敗的優先跑（failed-first）
```

### B3. 登入 / Session

#### 互動式登入（warm-login）

```powershell
python tools/warm_login.py
# 或
python tools/run.py --warm-login
```

- 跳出瀏覽器讓你手動登入（含 CAPTCHA）
- 完成後 session 存到 `.auth/admin.json` + `.auth/admin.session.json`
- 後續 pytest 自動重用，**直到 session 失效**才需重跑

#### 切換帳號

```powershell
python tools/warm_login.py --role admin   # 預設
python tools/warm_login.py --role <key>   # config.accounts 中其它 key
```

#### Session debug（少用）

```powershell
# session 失效時先看儲存內容對不對
python tools/dump_storage_now.py
# 在登入後即時 dump runtime storage（找 auth token 真正位置）
python tools/dump_runtime_storage.py
```

#### 冒煙驗證（session 是否還活著）

```powershell
python tools/run.py --smoke
```

### B4. 頁面探索 / 規格取材

#### 探索頁面 selector

```powershell
# 列出可見的 input placeholders / button texts / role 標記
python tools/explore_page.py /entry/<path>

# headless 模式（CI / 不開瀏覽器）
python tools/explore_page.py /entry/<path> --headless

# 同時截圖
python tools/explore_page.py /entry/<path> --shot snap.png
```

#### 從 PDF 抽規格文字

```powershell
# 單頁
python tools/extract_pdf_text.py --pages 17

# 多頁 / 範圍
python tools/extract_pdf_text.py --pages 17,18,19
python tools/extract_pdf_text.py --pages 17-20

# 輸出到檔案
python tools/extract_pdf_text.py --pages 17 --out spec.txt

# 換 PDF
python tools/extract_pdf_text.py --pdf 自訂.pdf --pages 1-3
```

預設 PDF：`input/需求規格.pdf`（user 本機自行命名 / 放置）。

### B6. 一些常見組合

#### 「我要交付這週進度」

```powershell
python tools/warm_login.py             # 確保 session 新鮮
pytest --shot=always                   # 全 case + 全截圖
python tools/md_to_docx.py             # 產 docx
# → reports/<最新 rid>_run/docx/*.docx
```

#### 「測一半被打斷，明天接續」

```powershell
python tools/warm_login.py             # session 已過期就重登
pytest --lf                            # 只重跑上次紅的 case
```

#### 「我想試一個新工項該怎麼起步」

```powershell
python tools/explore_page.py <該工項頁面 path>     # 先看頁面結構
python tools/extract_pdf_text.py --pages <頁碼>    # 抽規格文字
# 接著進 specs/ 寫 AC，再進 tests/ 寫 test_*.py
```
