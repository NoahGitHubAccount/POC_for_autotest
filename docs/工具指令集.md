## 1. 產 Word 報告

### 標準流程：跑測試 → 出 docx

```powershell
# 步驟 1：跑測試（每個 case 都截圖）
pytest --shot=always

# 步驟 2：把 reports/ 最新 run 整併成 docx
python tools/md_to_docx.py
```

輸出：`reports/<rid>_run/docx/<rid>_run_測試報告.docx`（單份檔，所有工項報告整併）。

### 指定特定 run

```powershell
# 用 run_id（reports/ 下的目錄名前綴）
python tools/md_to_docx.py 20260506_2243_run

# 或只給時間部分也可
python tools/md_to_docx.py 20260506_2243
```

### 沒裝 python-docx 時

```powershell
pip install python-docx==1.1.2
```

### 細節

- 截圖「嵌入」docx（不是參照），單檔可獨立流通
- `_summary.md` 在最前，其他工項報告依檔名排序，之間插分頁符
- 重跑 docx 不需要重跑 pytest——只要對應 run 的 `*.md` + `screenshots/` 還在就能轉

## 2. 跑測試

### 範圍選擇

```powershell
# 全部
pytest

# 單一工項目錄
pytest "tests/2-2-3 活動列表預設排序/" -v

# 多個工項
pytest "tests/2-2-3 活動列表預設排序/" "tests/2-2-4 活動列表欄位/" -v

# 用 wbs marker（如 2-2-2-B 全部）
pytest -m "wbs('2-2-2-B')" -v

# 只跑單一 test function
pytest "tests/2-2-2 搜尋功能/test_2_2_2_B_主辦單位活動日期活動名稱篩選.py::test_AC5_日期區間篩選" -v
```

### 截圖模式（pytest 自訂 option）

```powershell
pytest --shot=failed_only   # 預設：只失敗拍
pytest --shot=always        # 每個 case 都拍（產交付 docx 時用這個）
pytest --shot=off           # 完全不拍
```

### 失敗時看更多

```powershell
pytest -v --tb=short        # 簡短 traceback
pytest -v -x                # 第一個失敗就停
pytest -v --lf              # 只重跑上次失敗（last-failed）
pytest -v --ff              # 失敗的優先跑（failed-first）
```

## 3. 登入 / Session

### 互動式登入（warm-login）

```powershell
python tools/warm_login.py
# 或
python tools/run.py --warm-login
```

- 跳出瀏覽器讓你手動登入（含 CAPTCHA）
- 完成後 session 存到 `.auth/admin.json` + `.auth/admin.session.json`
- 後續 pytest 自動重用，**直到 session 失效**才需重跑

### 切換帳號

```powershell
python tools/warm_login.py --role admin   # 預設
python tools/warm_login.py --role <key>   # config.accounts 中其它 key
```

### Session debug（少用）

```powershell
# session 失效時先看儲存內容對不對
python tools/dump_storage_now.py
# 在登入後即時 dump runtime storage（找 auth token 真正位置）
python tools/dump_runtime_storage.py
```

### 冒煙驗證（session 是否還活著）

```powershell
python tools/run.py --smoke
```

## 4. 頁面探索 / 規格取材

### 探索頁面 selector

```powershell
# 列出可見的 input placeholders / button texts / role 標記
python tools/explore_page.py /entry/evevent

# headless 模式（CI / 不開瀏覽器）
python tools/explore_page.py /entry/evevent --headless

# 同時截圖
python tools/explore_page.py /entry/evevent --shot snap.png
```

### 從 PDF 抽規格文字

```powershell
# 單頁
python tools/extract_pdf_text.py --pages 17

# 多頁 / 範圍
python tools/extract_pdf_text.py --pages 17,18,19
python tools/extract_pdf_text.py --pages 17-20

# 輸出到檔案
python tools/extract_pdf_text.py --pages 17 --out spec.txt

# 換 PDF
python tools/extract_pdf_text.py --pdf 自訂.pdf --pages 1-3
```

預設 PDF：根目錄的「高雄數位市民活動模組需求.pdf」。

## 6. 一些常見組合

### 「我要交付這週進度」

```powershell
python tools/warm_login.py             # 確保 session 新鮮
pytest --shot=always                   # 全 case + 全截圖
python tools/md_to_docx.py             # 產 docx
# → reports/<最新 rid>_run/docx/*.docx
```

### 「測一半被打斷，明天接續」

```powershell
python tools/warm_login.py             # session 已過期就重登
pytest --lf                            # 只重跑上次紅的 case
```

### 「我想試一個新工項該怎麼起步」

```powershell
python tools/explore_page.py <該工項頁面 path>     # 先看頁面結構
python tools/extract_pdf_text.py --pages <頁碼>    # 抽規格文字
# 接著進 specs/ 寫 AC，再進 tests/ 寫 test_*.py
```
